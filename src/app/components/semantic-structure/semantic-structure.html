Regla 6: Encabezados y Estructura Semántica

La estructura semántica proper proporciona contexto y navegación clara para todos los usuarios, especialmente para usuarios de lectores de pantalla.

Ejemplo Completo de Página Semántica

<header role="banner">
  <h1>Título del Sitio</h1>
  <nav role="navigation" aria-label="Navegación principal">
    <ul>
      <li><a href="#">Inicio</a></li>
      <li><a href="#">Servicios</a></li>
    </ul>
  </nav>
</header>

<main role="main">
  <h1>Título Principal de la Página</h1>
  <section>
    <h2>Sección Principal</h2>
    <article>
      <h3>Artículo Independiente</h3>
      <p>Contenido del artículo...</p>
    </article>
  </section>
</main>

<aside role="complementary" aria-label="Contenido relacionado">
  <h2>Información Adicional</h2>
  <p>Contenido complementario...</p>
</aside>

<footer role="contentinfo">
  <p>© 2024 Sitio Web. Todos los derechos reservados.</p>
</footer>

Validación Manual

1. Verificar Jerarquía de Encabezados

  • Un solo <h1> por página
  • Sin saltos de nivel (h1 ? h3)
  • Estructura lógica y anidada

2. Verificar Landmarks

  • Presence de roles semánticos
  • Labels descriptivos
  • Estructura lógica del documento

Testing con Lectores de Pantalla

NVDA/JAWS Navigation

  • 1 - 6 - Saltar a encabezados específicos

Referencias Rápidas

Elementos Semánticos HTML5

  • <header> - Cabecera
  • <nav> - Navegación
  • <main> - Contenido principal
  • <section> - Sección temática
  • <article> - Contenido independiente
  • <aside> - Contenido complementario
  • <footer> - Pie de página

Roles ARIA Principales

  • banner - Cabecera del sitio
  • navigation - Navegación
  • main - Contenido principal
  • complementary - Contenido secundario
  • contentinfo - Información del sitio
  • search - Función de búsqueda
  • form - Formulario
