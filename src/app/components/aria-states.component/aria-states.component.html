⚠️ Errores Comunes

❌ Uso Incorrecto

<div role="button" onclick="...">
  Haz clic
</div>

Problema: Div con role="button" no tiene comportamiento nativo de botón

✅ Uso Correcto

<button onclick="...">
  Haz clic
</button>

Solución: Usar elemento button nativo con comportamiento accesible

❌ Estados Desincronizados

<div aria-expanded="false">
  <!-- contenido visible -->
</div>

Problema: aria-expanded="false" pero el contenido está visible

✅ Estados Sincronizados

<div aria-expanded="false" hidden>
  <!-- contenido oculto -->
</div>

Solución: Estado ARIA coincide con estado visual real

🧪 Herramientas de Prueba

Chrome DevTools

Usa el panel Accessibility para inspeccionar árboles ARIA y propiedades computadas.

  1. Abre DevTools (F12)
  2. Ve al panel Accessibility
  3. Selecciona elementos para ver sus propiedades ARIA

axe DevTools

Extensión del navegador que detecta problemas de accesibilidad incluyendo errores ARIA.

  1. Instala la extensión axe DevTools
  2. Abre el panel axe en DevTools
  3. Ejecuta un análisis para encontrar problemas ARIA

NVDA/JAWS

Prueba con lectores de pantalla reales para verificar que los anuncios ARIA funcionan.

  1. Instala NVDA (gratuito) o JAWS
  2. Navega por los componentes interactivos
  3. Verifica que los estados se anuncien correctamente
